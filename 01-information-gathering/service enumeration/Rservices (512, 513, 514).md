**Overview:**
- R-Services are a legacy suite of Unix remote access tools operating over TCP/IP on ports 512, 513, and 514. Replaced by SSH due to transmitting all data, including credentials, in plaintext, making them trivially interceptable via MITM.
- Authentication relies on `/etc/hosts.equiv` and `.rhosts` files, which define trusted hosts and users. A matching entry in either file bypasses credential checks entirely, allowing unauthenticated access if misconfigured.
- Still encountered on commercial Unix systems (Solaris, HP-UX, AIX) during internal penetration tests.
---
## R-Commands

|Command|Daemon|Port|Description|
|---|---|---|---|
|`rcp`|`rshd`|514/TCP|Bidirectional file copy between hosts. Overwrites existing files without warning|
|`rsh`|`rshd`|514/TCP|Opens a remote shell without a login prompt. Validates via `hosts.equiv` and `.rhosts`|
|`rexec`|`rexecd`|512/TCP|Executes shell commands on a remote host. Requires credentials but transmits them unencrypted|
|`rlogin`|`rlogind`|513/TCP|Remote login to Unix hosts, similar to Telnet. Auth overridden by trusted file entries|

---
## Connection

```
rlogin <target> -l <user>
```

Attempt login to the remote host as the specified user. Succeeds without a password if the host or user is listed as trusted in `.rhosts` or `hosts.equiv`.

---
## Enumeration

```
rusers -al <target>
```

List all interactively logged-in users on the target over the network. Returns username, hostname, TTY, login time, idle time, and originating remote host.
