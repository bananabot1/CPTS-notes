**Overview:**
- NFS is a network file system protocol for Linux and Unix systems, serving the same purpose as SMB but using an entirely different protocol. NFS clients cannot communicate directly with SMB servers.
- Built on ONC-RPC/SUN-RPC, exposed on TCP and UDP ports 111 and 2049. Uses External Data Representation (XDR) for system-independent data exchange.
- NFS has no native authentication or authorization mechanism. All auth is delegated to the underlying RPC protocol options, making misconfigured exports highly dangerous.
- Key footprinting ports are 111 (RPC portmapper) and 2049 (NFS). Both should be targeted during enumeration.
- If `no_root_squash` is set on an export, files created by root retain UID/GID 0. This enables privilege escalation: upload a SUID shell to the NFS share, then execute it via SSH to read files or escalate as another user.

---

## Dangerous Settings

|Setting|Description|
|---|---|
|`rw`|Read and write permissions granted|
|`insecure`|Allows ports above 1024|
|`nohide`|Sub-mounted filesystems are exported independently|
|`no_root_squash`|Files created by root keep UID/GID 0|

---

## Enumeration

```
sudo nmap --script nfs* -sV -p111,2049 <target>
```

Run all NFS-related NSE scripts against the target. Returns available shares, export contents, and share stats.

---

```
showmount -e <target>
```

List all NFS exports available on the target.

---

```
mkdir target-NFS
sudo mount -t nfs <target>:/ ./target-NFS/ -o nolock
```

Mount the remote NFS share locally. `-o nolock` disables file locking, required when the target does not support it. If `root_squash` is set, root privileges will not carry over to the mounted share.

---

```
sudo umount ./target-NFS
```

Unmount the NFS share after enumeration.
