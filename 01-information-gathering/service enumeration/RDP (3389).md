**Overview:**
- RDP is Microsoft's proprietary GUI remote access protocol, operating over TCP port 3389. Transmits display and control commands encrypted over IP networks at the application layer.
- TLS/SSL has been supported since Windows Vista, but many systems still accept weaker RDP Security mode encryption. Certificates are self-signed by default, so clients cannot distinguish legitimate from forged certificates, making MITM feasible when NLA is not enforced.
- Network Level Authentication (NLA) requires the connecting user to authenticate before a full RDP session is established, reducing exposure. Enabled by default on Windows Server via Server Manager.
- RDP is installed and active by default on Windows Server, requiring no additional software. Reaching it from the internet requires NAT port forwarding and firewall rules permitting inbound connections on port 3389.
---
## Enumeration

```
nmap -sV -sC -p3389 --script rdp* <target>
```

Run all RDP NSE scripts. Returns NLA status, product version, and hostname.

```
rdp-sec-check.pl <target>
```

Identify RDP security settings unauthenticated via handshake analysis. Developed by Cisco CX Security Labs.

## Connection

```
xfreerdp3 /u:<user> /p:<password> /v:<target>
```

Initiate an RDP session from Linux using xfreerdp3.
