**Overview:**
- SNMP monitors and manages network devices (routers, switches, servers, IoT) over UDP port 161. Control commands are sent via agents, and unsolicited alerts are pushed from server to client as traps over UDP port 162.
- The MIB (Management Information Base) is a standardized text file defining all queryable objects on a device in a tree hierarchy. Each node is identified by an OID, a unique numeric sequence that maps to a specific piece of device information.
- Community strings act as plaintext passwords controlling read/write access to SNMP data. SNMPv1 and v2c transmit them unencrypted, making them interceptable on the wire. SNMPv3 adds authentication and encryption but sees slow adoption due to migration complexity.
- From an attacker's perspective, a readable community string (commonly `public`) exposes the full OID tree, which can include hostnames, interfaces, running processes, installed software, and network configuration.
---
## Dangerous Settings

|Setting|Description|
|---|---|
|`rwuser noauth`|Full OID tree read/write access with no authentication required|
|`rwcommunity <community string> <IPv4>`|Full OID tree access granted to any host sending the community string|
|`rwcommunity6 <community string> <IPv6>`|Same as `rwcommunity` but for IPv6|

## Enumeration

```
snmpwalk -v2c -c <community string> <target>
```

Walk the full OID tree using SNMPv2c and a known community string. Returns all queryable system information exposed by the agent.

```
onesixtyone -c /opt/useful/seclists/Discovery/SNMP/snmp.txt <target>
```

Brute-force valid community strings against the target using a wordlist. Use when the community string is unknown.

```
braa <community string>@<target>:.1.*
```

Bulk OID enumeration using `braa`. Faster than snmpwalk for large OID trees once a valid community string is known.
