**Overview:**
- WinRM is Windows' built-in remote management protocol, using SOAP over TCP port 5985 (HTTP) and 5986 (HTTPS). Must be explicitly enabled and configured on Windows 10 and later.
- Built on top of WinRS (Windows Remote Shell), which allows arbitrary command execution on the remote system and is included by default from Windows 7 onward.
- From Linux, Evil-WinRM is the standard pentesting client for interacting with WinRM, providing an interactive shell with built-in support for file transfer, script loading, and pass-the-hash.
---
## Connection

```
evil-winrm -i <target> -u <user> -p <password>
```

Connect to the target WinRM service and drop into an interactive shell using Evil-WinRM.
