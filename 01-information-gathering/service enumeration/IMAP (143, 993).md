**Overview:**
- IMAP allows online email management directly on the server, including folder structures and multi-client synchronization. Unlike POP3, emails stay on the server and stay in sync across all clients simultaneously.
- Operates on port 143 by default. Encrypted sessions use SSL/TLS on port 993. Without encryption, all commands, credentials, and email content are transmitted in plaintext.
- SMTP handles outgoing mail. Integration with IMAP is achieved by copying sent emails into an IMAP folder, giving all clients access to sent mail regardless of which device was used.

## Dangerous Settings

|Setting|Description|
|---|---|
|`auth_debug`|Enables full authentication debug logging|
|`auth_debug_passwords`|Logs submitted passwords and their hashing scheme|
|`auth_verbose`|Logs failed authentication attempts and reasons|
|`auth_verbose_passwords`|Logs passwords used during authentication, optionally truncated|
|`auth_anonymous_username`|Sets the username for ANONYMOUS SASL mechanism logins|

---
## Enumeration

```
curl -k 'imaps://<target>' --user <user>:<password>
```

Authenticate to the IMAP server over SSL and list available mailboxes. `-k` skips certificate validation.

```
openssl s_client -connect <target>:993
```

Open a raw SSL connection to the IMAP service for manual interaction and certificate inspection.

## IMAP Commands

|Command|Description|
|---|---|
|`1 LOGIN <user> <password>`|Authenticate to the server|
|`1 LIST "" *`|List all mailbox folders|
|`1 SELECT INBOX`|Select a mailbox to access its messages|
|`1 UNSELECT INBOX`|Exit the currently selected mailbox|
|`1 FETCH <ID> all`|Retrieve all data for a specific message|
|`1 CREATE "INBOX"`|Create a new mailbox folder|
|`1 DELETE "INBOX"`|Delete a mailbox folder|
|`1 RENAME "ToRead" "Important"`|Rename a mailbox folder|
|`1 LSUB "" *`|List subscribed/active mailbox folders|
|`1 CLOSE`|Expunge messages flagged as deleted|
|`1 LOGOUT`|Close the connection|
